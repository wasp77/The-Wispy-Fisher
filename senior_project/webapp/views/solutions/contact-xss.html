Contact Page XSS

The contact page is vulnerable to persistent XSS. If you leave some feedback, you will see that it is immediately returned in the "Check Out What Others Said" section. If you look at the source code, you will see that your feedback is returned in a span element. However when trying to execute the standard XSS payload </span><script>alert("XSS")</script> you will see

This alert means that something in our payload is getting blocked by a filter. If you are diligent when looking at the source code, you will see that the filter is checking for the keyword "script." Therefore, in order to trigger the XSS we need to use an alternative payload like </span><img src="x" onerror=alert("XSS")>. Now, whenever you visit the page, you will see the XSS trigger.

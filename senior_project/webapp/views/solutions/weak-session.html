Weak Session Token

After signing in you will see a session token saved in your cookies . To test for weaknesses in this token create two separate accounts and using an incognito window or a separate browser sign into both accounts. Next, take the two session tokens, and base64 decode them using an online decoder. You will get something like {"secret":"wasp","id":"3"} where the "wasp" is a secret string and "id" is your ID. Now if you swap in the ID value of your other account and base64 encode you can use this new session token in place of your previous one, and you will see that you are logged into the other account. To change your session token use your browser's JavaScript console as described in the guide. We can up the severity of this vulnerability replacing the ID value with 1 and log yourself into the administrator account.

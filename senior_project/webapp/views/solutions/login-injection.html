Login SQL Injection

The login page has a SQL injection vulnerability in the login functionality. This vulnerability can be leveraged to log in as a user only knowing their username. You can test this using your account; however to showcase how severe these vulnerabilities are you can use it to login into the admin account. The problem occurs because the server is just checking that the database query for the password is valid, therefore if we craft a payload as described in the guide's SQL injection section ' or '1'='1 and enter this in the password field, we can see the login works. It works because the payload creates a SQL statement that is always true. Now, all we have to do to sign into the administrative account is to determine the administrator's username. Conveniently, they left a comment on the login page
so to complete the full exploit enter "admin" as the username and the payload for the password and see yourself logged in as the site administrator.
